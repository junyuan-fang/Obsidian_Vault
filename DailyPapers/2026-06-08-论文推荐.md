### 1. PiL-World: A Chunk-Wise World Model for VLA Policy-in-the-Loop Evaluation
- **作者**: Chong Ma, Taiyi Su, Jian Zhu, Jianjun Zhang, Zitai Huang, Yi Xu, Hanli Wang
- **机构**: 同济大学，AIRC 美的研究院
- **链接**: [arXiv](https://arxiv.org/abs/2606.05773) | [PDF](https://arxiv.org/pdf/2606.05773)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[PiL-World]]

![](https://arxiv.org/html/2606.05773v1/Figures/Figure1.png)

- **核心方法**: 给 [[VLA]] 做闭环评估的 chunk-wise [[World Model]]。输入：当前 obs + VLA 输出的 action trajectory；输出：多视角 future obs 视频。关键设计：(1) **chunk-wise** 与 VLA 的 action chunking 一致，每个 chunk 内部 closed-loop；(2) **action-derived visual control** 用 head-view 机器人运动 + latent history 做条件，提升 fidelity；(3) **joint prediction** 多视角互补 obs；(4) 训练时**显式喂失败轨迹**做 imagined rollout 真实分布对齐。交替执行 VLA inference + WM prediction 实现完整闭环评估，不需每步真机。
- **对比方法/Baselines**: 与 open-loop video WM (如 [[Ctrl-World]]) 对比；评估指标用真实成功率 SR 与 WM 估算 SR 的误差。
- **借鉴意义**: 这是 [[τ0-WM]] / [[OSCAR]] 之后第三块拼图 — Critic + Cosmos-Predict + 闭环评估三件套合体。对在 Figure / 银河 / 智元 / Apptronik 这种部署密度下，"不要每次都真机测"刚需。
- **锐评**: 真东西。问题点得准 — open-loop WM 评估在 chunked VLA 时代根本不能用，因为 action chunk 之间的 condition 链是闭环的。**误差从 63.2% → 12.0%** 这个数字硬得能砸人。"显式喂失败轨迹"和罗剑岚 [[τ0-WM]] "Critic 吃失败数据" 哲学同根，不是巧合，是这条路线在收敛。唯一遗憾：只在 3 个真实双臂任务上做，没扩到 humanoid 全身、没在 Cosmos / GR00T 这种更大 backbone 上 stress test。🔥
- **关联笔记**: 跟 [[τ0-WM]] "Critic + WM 闭环评估" 主线同根；与 [[OSCAR]] "Cosmos-Predict + 2D 骨架 virtual evaluation" 互补。

### 2. World-Language-Action Model for Unified World Modeling, Language Reasoning, and Action Synthesis
- **作者**: Yi Yang, Zhihong Liu, Siqi Kou, Yiyang Chen, Yanzhe Hu, Jianbo Zhou, Boyuan Zhao, Zhijie Wei, Xiao Xia, Xue Li 等
- **机构**: SJTU, NJUPT
- **链接**: [arXiv](https://arxiv.org/abs/2606.05979) | [PDF](https://arxiv.org/pdf/2606.05979)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[WLA]]

![](https://arxiv.org/html/2606.05979v1/x1.png)

- **核心方法**: 提出 **WLA**（World-Language-Action）作为新一类 embodied foundation model。输入：文本指令 + 图像 + 机器人状态；输出：联合预测文本子任务、subgoal 图像、机器人动作。把 [[WAM]]（World-Action Model）的"看视频学世界"和 [[VLA]] 的"语言推理 + 长时序任务"合体，**用 AR Transformer 单 backbone** 取代 [[VLA]] 主流的 bidirectional diffusion Transformer。在 RoboTwin 和 LIBERO 上 demo OOD 泛化。
- **对比方法/Baselines**: 借鉴 [[WAM]] + [[VLA]] 范式；可能对位 [[π0]] / [[OpenVLA]] / [[GR00T]] 系列；benchmark 含 [[RoboTwin]]、[[LIBERO]]。
- **借鉴意义**: NVIDIA 主导的 [[GR00T]] N1.5 是 VLA + WAM 双 head，但还没真正"统一" — WLA 这种"一个 AR backbone 同时输出子任务文本 + subgoal 图像 + 动作"才是真正的 unified embodied foundation model 形态。
- **锐评**: 这个标题挂得大，但方向是对的。"AR 取代 diffusion VLA backbone" 是对 [[π0]] / [[GR00T]] 主流路线的正面挑战。问题：(1) **AR 推理慢**这条没解决，对实时 control 是硬伤；(2) 只在 LIBERO + RoboTwin 这两个相对小规模 benchmark 上 demo，跟 Figure / 银河真机部署的鸿沟没补；(3) "WLA-0" 这种命名让人怀疑还有 WLA-1 在憋大招。但 Unified 这个路线本身值得跟。👀

### 3. DexFuture: Hierarchical Future-State Visuomotor Targeting for Bimanual Dexterous Tool Use
- **作者**: Runfa Blark Li, Kuang-Ting Tu, Nikola Raicevic, Dwait Bhatt, Xinshuang Liu, Keito Suzuki, Ki Myung Brian Lee, 等
- **机构**: UC San Diego
- **链接**: [arXiv](https://arxiv.org/abs/2606.05699) | [PDF](https://arxiv.org/pdf/2606.05699)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[DexFuture]]

![](https://arxiv.org/html/2606.05699v1/x1.png)

- **核心方法**: 解决双手灵巧工具使用中的"未来参考轨迹"难题。Hierarchical 设计：(1) 用 **DexWM**（自研 dexterous world model）在 latent space 生成 future state target，**不依赖 demonstration 的特权状态**；(2) [[PhysGraph]] 物理一致性约束；(3) 上层用 [[CEM]] 做 target-conditioned 规划；(4) 用 [[AdaLN]] 做条件融合。在 OakInk2 这种 contact-rich 数据上训。强调"非 demo-conditioned + 不需要 slow counterfactual planning"。
- **对比方法/Baselines**: 对位现有 future-action-conditioned WM 路线（如 [[UniSim]] / [[Genie]] 风格做规划用），强调"上层 hierarchical targeting + 下层 DexWM"分离的效率优势。
- **借鉴意义**: 双手灵巧 + 工具使用是 humanoid 商用化里最难的一块（Figure 还在 long-horizon picking 上挣扎），这种 hierarchical future-state targeting 给 humanoid manipulation 提供新的训练范式。
- **锐评**: 把"future-state targeting"从 vision-only 上升到双手 contact-rich 是有诚意的。**DexWM 独立成块**这个 design 关键 — 跟 [[PiL-World]] 一脉相承，WM 不必是"action 生成器"，可以是"future state 生成器"。问题：(1) OakInk2 是仿真数据，real-world 工具使用泛化没看到；(2) "Hierarchical Future-State" 这种 abstraction 听起来漂亮但通常 ablate 一下就发现 hierarchy 的贡献其实很有限；(3) 摘要里 PhysGraph 和 AdaLN 这种 buzzword 堆得密，看 ablation 才知道是不是真起作用。🔥

### 5. TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies
- **作者**: Dong Jing 等
- **机构**: 中国人民大学，UNC
- **链接**: [arXiv](https://arxiv.org/abs/2606.06491) | [PDF](https://arxiv.org/pdf/2606.06491)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2606.06491v1/x1.png)

- **核心方法**: 解决 [[VLA]] 只从演示继承单一固定速度的问题。关键观察：**每步预测 action 的 magnitude 已经隐含速度**。提出 **VSTA**（Variable-Speed Trajectory Augmentation）数据增广 + speed token 控制接口，让一个策略可以在高风险接触段慢下来，低风险 transit 段加速。在 [[LIBERO]] 上验证。
- **对比方法/Baselines**: 对比 model compression / KV-cache reuse / RL 加速派系；这些只能"切换"到另一个固定速度，没法连续可控。
- **借鉴意义**: 工业部署最现实的痛点。Figure BotQ 工厂如果每个 VLA 模型都要"为速度重训"，运营成本爆炸。
- **锐评**: 观察很巧妙 — "action magnitude = 速度"这个直觉早就有人提过，但真把它做成"训练时增广 + 推理时 token 控制"还是这篇先到。问题：(1) 摘要没说接触段慢下来时**力控**怎么协同（光慢不一定安全）；(2) 只在 LIBERO 上测，contact-rich 真机怎么样要看正文。👀
- 💡 **想精读？** 运行：`读一下 TempoVLA`

![](https://arxiv.org/html/2606.06323v1/Figures/front_figure.png)

- **核心方法**: 用 [[VLM]] 把演示轨迹切成 transit / contact 段，对 transit 段做大幅 downsampling，对 contact 段保留 frame rate，再训"faster-than-demo" policy。对位 [[DemoSpeedup]] 和 [[AWE]]（Automatic Waypoint Extraction）。
- **对比方法/Baselines**: 对比 uniform downsampling、[[DemoSpeedup]]、[[SAIL]]、[[AWE]]。
- **借鉴意义**: 跟 [[TempoVLA]] 是同一天里出现的"机器人能跑多快"双胞胎论文，但思路不同 — VOLT 在数据层切，TempoVLA 在 action magnitude 层切。两个可以叠加。
- **锐评**: VLM 切轨迹这种 trick 已经在多篇论文里见过（AWE 系），创新点在 "vision + language 双通道分段判定"。问题：(1) VLM 调用成本高，工业部署不友好；(2) "VLM 判 transit/contact" 在 contact 模糊的任务（如 deformable）容易翻车。👀
- 💡 **想精读？** 运行：`读一下 VOLT`

### 13. ActiveMimic: Egocentric Video Pretraining with Active Perception
- **作者**: Xingyao Lin 等
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2606.06194) | [PDF](https://arxiv.org/pdf/2606.06194)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2606.06194v1/x1.png)

- **核心方法**: 解决"用 egocentric human video 预训的 policy 不如 robot data 预训"问题。核心 insight：human 头部在 manipulation 时会主动 reposition，过去 pipeline 把这种 camera motion 当噪声丢掉了。ActiveMimic 从单 body-worn RGB 恢复**同步的 camera + wrist 轨迹**，把 camera motion 作为 viewpoint 控制信号一起预训。SFT + wrist branch。
- **对比方法/Baselines**: 对比 [[MotoVLA]] / [[VGGT]] 等 egocentric video pretraining 方案。
- **借鉴意义**: 给 [[GR00T]] / [[π0]] 这种 VLA 大模型预训数据池子加新 modality（active perception）。
- **锐评**: 这个 insight 是真的 — "head motion 是 active perception signal 不是噪声" 这件事过去 egocentric pretraining 普遍忽略。问题：(1) 单 body-worn RGB 恢复 camera + wrist 轨迹的精度上限有限；(2) 真到 humanoid policy 上 transfer，wrist 自由度和真机器人的差异要怎么处理摘要没说。👀
- 💡 **想精读？** 运行：`读一下 ActiveMimic`

### 14. MiTaS: Multi-Resolution Tactile Imitation Learning for Contact-Rich Robotic Manipulation
- **作者**: Rickmer Krohn 等
- **机构**: TU Darmstadt, Hessian AI, Robotics Institute Germany
- **链接**: [arXiv](https://arxiv.org/abs/2606.06281) | [PDF](https://arxiv.org/pdf/2606.06281)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2606.06281v1/figures/hero_mitas.png)

- **核心方法**: 多分辨率触觉 imitation learning 框架。多个异构 tactile sensor（不同时间分辨率）通过 modality-specific conv stem + transformer fusion 合并。架构含 [[ViT]] + [[AdaLN]] + [[DiT]] 风格 fusion，搭配 [[GelSight]] 这种主流触觉传感器。
- **对比方法/Baselines**: 对位单 tactile + visuomotor IL；与 [[Diffusion Policy]] 兼容。
- **借鉴意义**: contact-rich manipulation（双手装配 / 拧螺丝 / 推扣）是 humanoid 痛点。多分辨率触觉是给 fine grain control 的必经工程。
- **锐评**: TU Darmstadt 做触觉一向扎实。"multi-resolution" 这种说法听起来漂亮，但 ablation 里通常会发现"高分辨率单 sensor 就够了"。要看正文里多 sensor 实际带来的 SR 增量。👀
- 💡 **想精读？** 运行：`读一下 MiTaS`

### 15. ADWM: Autoregressive Diffusion World Models for Off-Policy Evaluation of LLM Agents
- **作者**: Kaixuan Liu, Guojun Xiong, Weinan Zhang, Shengpu Tang
- **机构**: 上海交大, Emory University 等
- **链接**: [arXiv](https://arxiv.org/abs/2606.05558) | [PDF](https://arxiv.org/pdf/2606.05558)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2606.05558v1/x1.png)

- **核心方法**: 给 [[LLM]] agent 在多轮交互环境的 OPE（off-policy evaluation）。学一个 latent **AR diffusion WM** 模拟"环境对评估策略如何响应"，不真跑环境。与现有"diffusion 一次性生成整段轨迹"区别 — ADWM autoregressive 一步步生成。在 [[HotpotQA]] / [[WebShop]] / [[ScienceWorld]] 上 demo。
- **对比方法/Baselines**: 对位 model-based OPE / [[FQE]] / [[IS]] / [[DR]]。
- **借鉴意义**: 给 LLM agent eval 提供 sim 替代。robotics 角度看，"AR diffusion WM 一步步 rollout"哲学跟 [[PiL-World]] 异曲同工 — 都是为了避免一次性 rollout 整段误差爆炸。
- **锐评**: 不是直接 robotics 论文但**思路对 robotics WM 有借鉴**。问题：LLM agent 环境 transition 远比物理环境简单，这种 AR diffusion WM 的 fidelity 要求低很多。看能不能迁移到 embodied 更复杂的 transition。👀
- 💡 **想精读？** 运行：`读一下 ADWM`

### 16. DoPr: Double Preconditioning — Optimization for Test-Time Performance, not Validation Loss
- **作者**: Thomas T. Zhang, Alok Shah, Victor Zhang, Nik Matni 等
- **机构**: U Penn, CMU, Amazon FAR
- **链接**: [arXiv](https://arxiv.org/abs/2606.06418) | [PDF](https://arxiv.org/pdf/2606.06418)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2606.06418v1/figs/figs_ttf/flowchart1.png)

- **核心方法**: 提出 "test-time feedback (TTF)" 现象：单步预测 loss（L2 / cross-entropy）训的模型在 autoregressive rollout / flow matching / **robot policy** 部署时性能与 validation loss 错开。DoPr = double preconditioning 优化器（[[AdamW]] / [[KFAC]] 改良），针对 TTF 设计，理论 + 实验验证。
- **对比方法/Baselines**: 对位 [[AdamW]]、[[KFAC]]、[[SFT]]、[[LoRA]] 优化器；用 [[GP]] / [[GD]] 框架做理论分析。
- **借鉴意义**: robot policy learning 是文中明确举例的"TTF 受害者"。如果 DoPr 在 robot policy 训练里真带来 task SR 提升，是个无脑可换的优化器。
- **锐评**: 理论 paper 通常 over-promise，"TTF 现象"听起来像 covariate shift 换个名字。但 U Penn + CMU + Amazon FAR 这种组合做理论 + 实验都靠谱。robotics 角度看：等谁在真 manipulation policy 上做对比再说，光看摘要不足以判断对实际 SR 的增量。👀
- 💡 **想精读？** 运行：`读一下 DoPr`
