### 1. minWM: A Full-Stack Open-Source Framework for Real-Time Interactive Video World Models
- **作者**: 待补全（HF Trending u=55）
- **机构**: Tsinghua, HKUST
- **链接**: [arXiv](https://arxiv.org/abs/2605.30263) | [PDF](https://arxiv.org/pdf/2605.30263)
- **来源**: 🔥 HF Trending，⬆️ 55
- 📒 **笔记**: [[minWM]]

- **核心方法**: 把多步双向 [[diffusion]] T2V/TI2V 模型转成 [[camera-controlled]] 实时交互 video WM 的全栈框架。包含 (a) [[DMD]] (Diffusion Matching Distillation) 把多步蒸馏到 [[AR]] 单步生成；(b) [[ODE]] solver 加速；(c) [[SpatialVid]] 数据集 + [[WorldPlay]] 训练 pipeline；(d) GitHub 完整代码 + 权重。输入：相机控制信号 + 文本提示；输出：实时交互视频帧。
- **对比方法/Baselines**: 主要对比 [[Cosmos]] / [[DreamerV3]] 这类 WM 路线，但走的是"video 生成派"而非传统 latent state 派。
- **借鉴意义**: 对做具身仿真、Sim2Real video 数据生成、训练数据飞轮的人价值很大 — 实时 + 交互 + 开源三件套同时满足的 WM 工程化标杆。
- **锐评**: 真东西。"实时交互 video WM 开源全栈" — 这话说了一年终于有人交卷。数据 + pipeline + 权重 + GitHub 全开，比 NVIDIA Cosmos 半开源态度厚道得多。问题在于：T2V/TI2V 蒸馏到单步后画质损失多少？SpatialVid 数据 domain 多窄？等代码跑过再说。但光是"开源 + 全栈"就够拿 🔥 了。

### 2. NVIDIA OmniDreams: Real-Time Generative World Model for Closed-Loop Autonomous Vehicle Simulation
- **作者**: NVIDIA
- **机构**: NVIDIA
- **链接**: [arXiv](https://arxiv.org/abs/2606.03159) | [PDF](https://arxiv.org/pdf/2606.03159)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[OmniDreams]]

- **核心方法**: 用 generative WM 给自动驾驶做闭环仿真 — 自车决策反过来影响场景演化。重点解决长尾场景：[[generative]] WM 可以生成 corner case；闭环可以测策略在 OOD 状态的行为。摘要未透露具体架构（推测 video diffusion + action conditioning）。
- **对比方法/Baselines**: 自动驾驶 Sim 主流是 [[CARLA]] / [[NuSim]] / Waymo simulator，NVIDIA 走"全生成"路线对位 Wayve [[GAIA-1]] / [[GAIA-2]]。
- **借鉴意义**: 对做 Sim2Real、policy evaluation、long-tail safety 的人是大事 — 闭环 + 生成式 + 实时三件套对位 Wayve 的护城河。
- **锐评**: NVIDIA 终于在自动驾驶 simulator 这条线上拿出 generative 答卷。但是摘要太薄，没看到 fidelity / latency 数字。这种 closed-loop generative 摘要里全是 "remains a critical bottleneck" 这种话术，等代码看。叫 OmniDreams 听起来像 Cosmos 的车版本，做 Wayve GAIA 的对位。方向对，但 NVIDIA 这种官方 paper 给的细节常常不够。

### 3. GN0: Toward a Unified Paradigm for Generation, Evaluation, and Policy Learning in Visual-Language Navigation
- **作者**: 待补全
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2606.03682) | [PDF](https://arxiv.org/pdf/2606.03682)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[GN0]]

- **核心方法**: 在 [[VLN]] 任务里把数据生成、评估、策略学习放进一个统一框架。摘要点出 VLN 长期受限于数据可得性 + 质量。GN0 提出统一 paradigm。具体怎么 unify 摘要没说清楚，但"生成 + 评估 + 策略"这种 framing 直接对位昨天讨论的罗剑岚 Critic 主线。
- **对比方法/Baselines**: VLN 主流 [[R2R]] / [[RxR]] / [[REVERIE]] 系；评估方向有 [[NavRL]] / [[ETPNav]]。
- **借鉴意义**: 跟昨天罗剑岚 τ0-WM "Critic + WM + 策略统一" 的 narrative 一摸一样，只是场景换到 navigation。值得看是不是真做到了。
- **关联笔记**: 与昨日讨论的"VLA wrapper 三件套" + 罗剑岚 Critic 主线呼应。
- **锐评**: 标题写得野心很大 — "Toward a Unified Paradigm"。但摘要里全是 "we propose"，没有数字、没有 benchmark、没有对位的 baseline 名字。GN0 这名字像是 GPT-N 这种命名套路。要么是真东西，要么是 incremental 包装。从 VLN 角度看：把生成数据、评估、策略放一个框架确实是这一波 RL × WM 的正确方向。但执行得好不好等代码。
