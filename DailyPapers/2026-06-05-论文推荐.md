### 1. Cosmos 3: Omnimodal World Models for Physical AI
- **作者**: NVIDIA
- **机构**: NVIDIA
- **链接**: [arXiv](https://arxiv.org/abs/2606.02800) | [PDF](https://arxiv.org/pdf/2606.02800)
- **来源**: 🔥 HF Trending，⬆️ 64
- 📒 **笔记**: [[Cosmos3]]

- **核心方法**: NVIDIA Cosmos 系列第三代，**mixture-of-transformers omnimodel** 一栈处理 text + images + video + ambient sound + actions。三角色：(a) [[VLM]] 做多模态推理；(b) **World Model** 做物理环境仿真；(c) **WAM Backbone** 训练 task-specific 机器人模型。4B-14B 参数三档（Nano / Super / Ultra）。训练数据：超 **2000 万小时**机器人 + 自动驾驶视频（DGX Cloud 上数千张 GPU 训）。
- **对比方法/Baselines**: [[Cosmos]] 1 / [[Cosmos]] 2（自家迭代）；外部对位 [[GAIA-2]]、Wayve 系列、[[Genie 2]]、Sora 类视频生成派；机器人侧对位 [[DreamerV3]] / 传统 WM。
- **借鉴意义**: 整个 NVIDIA Physical AI 生态（GR00T N / Isaac / Alpamayo）背后的模型底座论文。如果你做 humanoid / robotics foundation model，这是 2026 H1 必引。
- **锐评**: 6/1 Computex 上 Jensen 嘴里的"Cosmos 3 三角色"具体规格终于挂到 arXiv。三角色路线（VLM + WM + WAM backbone）是正确范式，但是不是真做到"五种模态一栈处理"还要看具体数字。NVIDIA 的 paper 习惯是细节给的不够，规格图很漂亮但 ablation 不齐。等代码、等权重发布。无论如何，6/1 这一波 NVIDIA Physical AI 全套（Cosmos 3 + GR00T N1.5 + GR00T Reference Humanoid = Unitree H2 Plus）的 "模型层"就这一篇。
- 💡 **想精读？** 运行：`读一下 Cosmos 3`

### 2. GRAIL: Generating Humanoid Loco-Manipulation from 3D Assets and Video Priors
- **作者**: NVIDIA + UCLA 团队
- **机构**: NVIDIA, UCLA
- **链接**: [arXiv](https://arxiv.org/abs/2606.05160) | [PDF](https://arxiv.org/pdf/2606.05160)
- **来源**: 🔥 HF Trending，⬆️ 5
- 📒 **笔记**: [[GRAIL]]

![](https://research.nvidia.com/labs/dair/grail/)

- **核心方法**: 给 humanoid 生成 loco-manipulation 行为。输入：3D 资产（物体模型）+ video priors（视频先验）。架构涉及 [[HOI]] (Human-Object Interaction) 生成 + [[SMPL]] 人体参数化 + Task-General 控制。NVIDIA 项目页公开。
- **对比方法/Baselines**: HOI 主流（[[Object-Centric]] 路线 + Human Motion 路线）；Loco-Manipulation 对位 [[HumanPlus]] / [[HEX]]。
- **借鉴意义**: 给 humanoid loco-manipulation 数据生成方向新思路 — "3D 资产 + 视频先验 → 动作数据"是 sim-to-real data pipeline 关键一环。与 5/30 用户讨论的"评价驱动" + 数据飞轮配合。
- **关联笔记**: 与昨天 [[Humanoid-GPT]]（2B 帧 motion 数据 GPT 范式）形成 "humanoid 数据 scaling 两件套"，加上 NVIDIA 24K trajectories 开源 = 三件套。
- **锐评**: NVIDIA + UCLA 出品，思路上接 HumanPlus 那条线但加了 3D 资产作 prior。SMPL 人体参数化 + HOI 生成是经典玩意儿，新点在 "task-general 控制 + 3D 资产 + video prior 同时"。问题：3D 资产从哪来？是手 build 还是用 PhysX-Omni 这种生成？视频 prior 是用 Cosmos 自家的还是外部？等正文。
- 💡 **想精读？** 运行：`读一下 GRAIL`

### 3. OVO-S-Bench: A Hierarchical Benchmark for Streaming Spatial Intelligence in Multimodal LLMs
- **作者**: 待补全
- **机构**: 清华大学 + 上海 AI Lab + 北航
- **链接**: [arXiv](https://arxiv.org/abs/2606.03890) | [PDF](https://arxiv.org/pdf/2606.03890)
- **来源**: 🔥 HF Trending，⬆️ 28

- **核心方法**: 评估 [[MLLM]] 在 **streaming spatial intelligence** 上的能力——空间理解从 single image / multi-view 进化到流式视频。覆盖空间推理 / 物体追踪 / 几何变化 / 视角变换四个维度。提到 [[RoboBrain2]] / [[InternVL]] / [[VST]] / [[FluxMem]] 等模型对比。
- **对比方法/Baselines**: 空间智能 benchmark 主流（image-level / multi-image / video 三档），OVO-S 把 streaming 单独拎出来做层次化评估。
- **借鉴意义**: 对做 embodied navigation / VLA spatial reasoning / autonomous driving 的人，streaming 是更接近真实部署的设置。值得收藏。
- **锐评**: 清华 + 上海 AI Lab + 北航三家联手做 benchmark，规格上肯定够（这种组合 paper 数据集 + 模型对比一般都做得细）。但 benchmark paper 本身没有方法创新，看你需不需要这类评估工具。值得收藏，不必精读。
