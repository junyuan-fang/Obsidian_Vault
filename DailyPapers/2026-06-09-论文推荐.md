### 1. STRIPS-WM: Learning Grounded Propositional STRIPS-style World Models from Images
- **作者**: 未知（论文 author 字段未抓取完整）
- **机构**: Worcester Polytechnic Institute, Department of Robotics Engineering
- **链接**: [arXiv](https://arxiv.org/abs/2606.06832) | [PDF](https://arxiv.org/pdf/2606.06832)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[STRIPS-WM]]

- **核心方法**: 给 long-horizon visual manipulation 做"用图像学**离散 propositional STRIPS-style** [[World Model]]"。关键设计：(1) 从图像中端到端提取**离散 STRIPS 命题谓词**（"on(A, B)"、"clear(C)"这种）；(2) 用 [[SAT]] / [[CP]] 求解器在符号空间做**长时序规划**；(3) [[FSQ]] (Finite Scalar Quantization) 做离散 latent 编码；(4) DinnerTable benchmark 上验证。
- **对比方法/Baselines**: 对位 continuous latent [[World Model]]（如 [[DreamerV3]] / [[GAIA-1]] 路线）；与神经符号 [[PDDL]] 派对比。
- **借鉴意义**: 给 long-horizon 任务规划提供"**离散符号 + 神经感知**" hybrid 路线。比起 continuous WM 一步步 rollout，符号空间规划在长时序更可解释 + 高效。
- **锐评**: 这是一个被低估了好几年的方向 — DeepMind 在 2018 RL 时代就玩过"learning STRIPS"，但当时图像太难端到端学。WPI 把 FSQ + SAT 结合上来是有诚意的工程。问题：(1) DinnerTable 这种 benchmark 偏 toy，真扔给 [[OakInk2]] / [[RoboTwin]] 这种复杂场景还能不能学到 sound 的 STRIPS predicates 要打问号；(2) "propositional 符号空间"对 deformable / contact-rich 任务是天然 mismatch；(3) Worcester Polytechnic 不是 mainstream 圈，论文影响力可能取决于代码 / 数据是否开源。但方向值得跟。🔥
- **关联笔记**: 与 [[World Model]] 概念体系互补，给 [[Cosmos3]] / [[PiL-World]] 这种 continuous WM 路线提供 hybrid 选项。

### 2. AdaWAM: Dreaming when Necessary — Advancing World Action Models with Adaptive Multi-Modal Reasoning
- **作者**: 未知（待补全）
- **机构**: 清华大学
- **链接**: [arXiv](https://arxiv.org/abs/2606.07089) | [PDF](https://arxiv.org/pdf/2606.07089)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[AdaWAM]]

- **核心方法**: 给 [[WAM]] (World Action Model) 加**自适应多模态推理 + 按需 dream**机制。Insight: WAM 不该每步都做 video prediction，简单步骤可以直接执行，长时序复杂步骤才"dream"。AdaWAM 学一个**何时调用 video prediction prior** 的 gating，简化 inference 同时保持 long-horizon 能力。Benchmark: [[LIBERO]] / [[RoboTwin]]，集成 [[VLA]] + VR。
- **对比方法/Baselines**: 对位现有 WAM（[[GR00T]] WAM head、[[π0]] WM 派、6/8 PiL-World、6/5 WLA）；都是"always dream" 范式。
- **借鉴意义**: WAM 推理成本是工业部署最大瓶颈之一（[[Cosmos3]] 14B 跑一次 rollout 成本可怕）。"按需 dream" 是把 WAM 从研究推到部署必经一步。
- **锐评**: 思路对 — "always dream" 是 WAM 的原罪，但实际工业部署根本承担不起每步 video gen。问题：(1) gating "何时 dream" 是个 meta-policy 学习问题，可能比 WAM 本身还难训；(2) 清华出 paper 普遍 ablation 详细但摘要藏数字，要等正文；(3) AdaWAM 这种命名跟 6/5 WLA 一样有"WAM-0 后面还有更大版本"的暗示。但作为 WAM 工业化的必经路线，值得跟。🔥
- **关联笔记**: 跟 [[PiL-World]] "chunk-wise WM" + [[Cosmos3]] "Omnimodal WM" + [[WLA]] "AR 统一 W+L+A" 同根 — WAM 系 2026 H1 的三大演化路径。
- 💡 **想精读？** 运行：`读一下 AdaWAM`

### 3. LARA: Latent Action Representation Alignment for Vision-Language-Action Models
- **作者**: 未知（待补全）
- **机构**: 未知（待补全）
- **链接**: [arXiv](https://arxiv.org/abs/2606.07100) | [PDF](https://arxiv.org/pdf/2606.07100)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[LARA]]

- **核心方法**: 用 **[[LAM]]** (Latent Action Model) 在 [[OXE]] (Open X-Embodiment) 跨数据集做 latent action 对齐，解决 "real-world robot action data 稀缺"。架构：[[DiT]] backbone + GPT-style decoder。先在大规模 OXE + Action 标注数据上学 latent action representation，然后用 latent 对齐做下游 [[VLA]] 训练。
- **对比方法/Baselines**: 对位现有 VLA 训练范式（[[OpenVLA]] / [[π0]] / [[GR00T]] N1.5）；借鉴 [[OXE]] 跨数据集预训。
- **借鉴意义**: VLA 跨 embodiment 泛化的核心问题就是 "action space 不一致"。LARA latent action alignment 是给"用 100 种 robot 数据训一个 VLA" 提供更通用的 representation 通道。
- **锐评**: latent action 不是新概念（[[GENIE]] 已经在视频生成端做过），把它正式用到 VLA 跨数据集对齐是合理推进。问题：(1) DiT + GPT 这种 backbone 组合现在很挤，真正区分度要看 ablation；(2) 摘要里 OXE 数据规模 / 下游 LIBERO SR 数字没给，光看摘要判断不了优势大小；(3) 真正能 zero-shot 跨新 embodiment 的 VLA 现在还没出现，LARA 是不是真做到这一步要等正文。🔥
- **关联笔记**: 与 [[OpenVLA]] / [[π0]] / [[WLA]] 同 VLA 路线；latent action 思路跟视频生成端 [[GENIE]] 同根。
- 💡 **想精读？** 运行：`读一下 LARA`

### 13. ActionMap: Robot Policy Learning via Voxel Action Heatmap
- **作者**: 待补全
- **机构**: Show Lab NUS, NVIDIA
- **链接**: [arXiv](https://arxiv.org/abs/2606.06904) | [PDF](https://arxiv.org/pdf/2606.06904)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: VLA 的 action decoder 长期"单点 prediction"未变。ActionMap 把它换成 **voxel action heatmap** — 把整个 3D 空间离散为 voxel，预测每个 voxel 的"是否为 action endpoint"的 heatmap。架构基于 [[OpenVLA]] + [[OFT]] (Offset Tokenization) 微调。Benchmark: [[LIBERO]] 真机。
- **对比方法/Baselines**: 对位 [[OpenVLA]] 单点 prediction decoder；与 [[π0]] / [[GR00T]] 比 decoder 设计差异。
- **借鉴意义**: VLA decoder 大改是 2026 H2 必经环节 — 单点 prediction 限制了 spatial reasoning。voxel heatmap 是 manipulation 端的对位 motion planning "occupancy grid" 思路。
- **锐评**: NUS Show Lab + NVIDIA 组合，工程上稳。voxel heatmap 决定计算量 + voxel 分辨率 trade-off — 分辨率高了内存爆，低了精度损失。OpenVLA 上微调是有保留，没说能不能 from scratch。👀
- 💡 **想精读？** 运行：`读一下 ActionMap`

### 15. GenPO++: Generative Policy Optimization with Jacobian-free Likelihood Ratios
- **作者**: 待补全
- **机构**: ShanghaiTech, 上海 AI Lab
- **链接**: [arXiv](https://arxiv.org/abs/2606.06967) | [PDF](https://arxiv.org/pdf/2606.06967)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: flow-based 生成式 policy 对 RL 很有吸引力（多模态 action distribution + 连续控制）。GenPO++ 提出 **Jacobian-free** likelihood ratio，避免 [[FPO]]-style 计算开销，让 [[PPO]] 风格更新可行。在 [[IsaacLab]] 上验证。
- **对比方法/Baselines**: 对位 PPO + Gaussian policy；flow-based RL 派 [[FPO]] / [[TV]]。
- **借鉴意义**: 现在 [[Diffusion Policy]] 路线占主导，但 diffusion 在 RL 训练中梯度估计困难。GenPO++ 给 flow-based RL 提供工程突破口。
- **锐评**: ShanghaiTech 出 RL 论文一向理论扎实。"Jacobian-free likelihood ratio"是真痛点 — flow-based RL 长期被这个卡。但 RL 论文标题挂 "++" 通常意味着 incremental over 自家上一代。👀
- 💡 **想精读？** 运行：`读一下 GenPO`

### 16. T-GMP: Terrain-conditioned Generative Motion Priors for Versatile and Natural Humanoid Locomotion
- **作者**: 待补全
- **机构**: 哈尔滨工业大学

- **锐评**: HIT 在四足 + humanoid loco 一向有作品。CVAE + LiDAR + AMP 配置经典，关键看 demo 的地形多样性是 sim 还是真机。👀
- 💡 **想精读？** 运行：`读一下 T-GMP`
